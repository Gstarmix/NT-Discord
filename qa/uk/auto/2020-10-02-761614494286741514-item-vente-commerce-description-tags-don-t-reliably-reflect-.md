# Item 'Vente'/'Commerce' description tags don't reliably reflect sell/trade restrictions (known issue)

**Q:** Some items can be sold to an NPC or exchanged even though their description says they shouldn't be (or vice versa) — is this a bug?

**A:** This was reported to staff as a potential bug. Players found that the 'Vente' (sale) and 'Commerce' (trade) description tags, which are supposed to indicate whether an item can be sold to NPCs or exchanged/traded, did not consistently match the item's actual behavior — some items stayed non-sellable/non-tradeable regardless of their tags. It's unclear whether this is an actual bug or just a mistranslation/mislabeling of the item description; the case was still under investigation.
