# Shell rune became character-bound without warning after maintenance

**Q:** After a recent maintenance, a shell that was applied to a dagger and previously did not make the weapon exclusive to the character now shows the weapon as exclusive. Is this a bug or intended, given there was no warning before it happened?

**A:** This is a known issue reported by more than one player: applying a shell used to leave the weapon tradeable/non-exclusive, but since a recent maintenance the same action now binds the weapon to the character, with no warning or confirmation popup (unlike the newer rune system, which does warn players before locking an item). Multiple players hit the same unexpected binding, so it is likely an unintended side effect rather than a documented change.
